# Arduino UDP Controller

Простой веб-сервер на Python + Flask, который позволяет отправлять UDP-команды на Arduino для управления робо-клешней («Открыть»/«Закрыть») через веб-интерфейс. Каждое действие логируется в базу данных SQLite с отметкой времени.

## Описание

Сервер запускается локально. С помощью кнопок можно открывать и закрывать клешню соотвественно. При нажатии кнопки бразуер отправляет запрос по адресу /action/on (открыть) или /action/off (закрыть) в зависимости от задачи. Flask-сервер ловит запросы и отправляет UDP-пакет на Arduino, делает запись в базу данных и, как по-умолчанию, возвращает текстовый ответ (строку "OK"). Этот ответ "OK" от сервера приходит обратно в браузер, но загружается не на главную страницу, а внутрь скрытого фрейма `hidden-form`. Для пользователя всё выглядит так: он нажимает на кнопку, пульт остается на экране, ничего не моргает и не перезагружается, а команда на Arduino уходит.

Все события записываются в файл `logs.db` и накапливаются — данные не перезаписываются при перезапуске сервера. Посмотреть этот файл можно по ссылке `Открыть журнал действий` `(http://127.0.0.1:5050/logs)`

## Требования

- Python 3.x
- Flask

## Установка

1. Убедитесь, что Python установлен:

   ```
   python --version
   ```
2. Установите Flask:

   ```
   pip install flask 
   ```

## Запуск

```
python server.py
```

После запуска сервер будет доступен по адресу: `http://127.0.0.1:5000`.

## Команды (маршруты)


| Адрес                         | Описание                                                           |
| ---------------------------------- | -------------------------------------------------------------------------- |
| `http://127.0.0.1:5000/`           | Главная страница с подсказкой                    |
| `http://127.0.0.1:5000/on`         | Открыть клешню (записать статус ON с текущим временем)        |
| `http://127.0.0.1:5000/off`        | Закрыть клешню (записать статус OFF с текущим временем)       |


## Файл логов

Все события сохраняются в `logs.db` рядом с `server.py`. Пример содержимого:

```
ID	Действие	Время
31	OFF	2026-06-12 23:43:14
30	OFF	2026-06-12 23:43:13
29	ON	2026-06-12 23:43:12
```

Файл создаётся автоматически при первом обращении к `/on` или `/off`.
